The Balanced Split
time limit per test
1 second
memory limit per test
256 megabytes
input
standard input
output
standard output

In the village of Bir Zayt, young Rami is learning clever ways to search through large tables of numbers. He believes that splitting the table into many parts will always make searching faster. But the village elder, Abu Yazan, explains that splitting into more parts also requires more comparisons per step, so the optimal choice of how many parts to split into is not obvious.

To settle the debate, Rami asks:

"For a table of size $$$n$$$, what is the best number of parts $$$k>1$$$ that minimizes the total search cost?"

The search cost of splitting into $$$k$$$ equal parts is modeled in worst case as

$$$$$$ C_k \cdot \log_k(n)$$$$$$

where $$$C_k$$$ is denoted as the number of comparisons required in one search step when splitting into $$$k$$$ parts.

Given $$$n$$$, determine the integer $$$k>1$$$ that yields the most efficient overall search cost.

Input

The first line contains a single integer $$$t$$$ $$$(1 \le t \le 10^5)$$$ — the number of test cases.

The first line of each test case contains one integer $$$n$$$ ($$$2 \le n \le 10^7$$$).

Output

Output a single integer $$$k$$$ — the optimal value of $$$k>1$$$ for which a $$$k$$$-ary search on a table of size $$$n$$$ achieves the most efficient search cost.If there are multiple answers, you should output the smallest.

Example

Input
3
4
5
8
Output
2 
2 
2
